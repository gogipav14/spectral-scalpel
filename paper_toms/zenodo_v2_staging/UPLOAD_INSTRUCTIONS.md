# Zenodo v2 upload instructions

The zip in this directory (`scalpel-toms-v2.zip`, 9.5 MB, 207 files)
is the fresh code archive to upload as **v2 of the existing concept
record** `10.5281/zenodo.19834321`.

Uploading as a new version (rather than a new record) preserves
the concept DOI, so older citations to v1 continue to resolve, and
the manuscript can cite the v2 version-DOI (which Zenodo assigns at
upload time and is the right identifier for an artifact-evaluated
submission).

## Step-by-step

1. **Log in to Zenodo** as the same account that owns the v1
   record. URL: <https://zenodo.org/account/login>.
2. Open the v1 record page: <https://zenodo.org/records/19834321>.
3. Click **"New version"** in the right-hand sidebar. Zenodo
   pre-fills the form with v1's metadata.
4. **Replace the files**: remove the existing `code.zip`, upload
   `scalpel-toms-v2.zip` in its place. Keep `CITATION.cff`,
   `LICENSE`, `README.md` as separate top-level files (Zenodo
   shows them alongside the zip); upload the fresh copies from
   this directory if you prefer (they match what's inside the zip).
5. **Update the title** to:
   `scalpel: A multi-backend FFT-NILT primitive for spectral slab PDE solvers, with class-dependent finite-precision auto-tuning`
   (matches the paper-1 TOMS title).
6. **Update the description** with the contents block from
   `paper_toms/zenodo_v2_staging/DEPOSIT_DESCRIPTION.txt` (see
   below).
7. **Add this manuscript as a related identifier**: type "is
   supplement to", DOI (TBD: ACM TOMS will assign one at acceptance;
   leave blank for now or use a placeholder URL to the manuscript
   PDF if Zenodo allows).
8. **Add keywords** (already in v1; verify): `scientific software`,
   `numerical inverse Laplace transform`, `spectral methods`,
   `GPU computing`, `multi-backend portability`, `reproducibility`,
   `Dubner-Abate inversion`.
9. **Verify version field reads `0.2.0`** (matches `pyproject.toml`
   and the `__version__` string in `scalpel/__init__.py`).
10. **Publish**. Zenodo assigns the new version-DOI; copy it.

## After upload — propagate the new DOI

Update four files in the repo with the new version-DOI Zenodo
returns (call it `10.5281/zenodo.NNNNNNN` below):

| File | Search → replace |
|---|---|
| `paper_toms/article.tex` | abstract Zenodo line + Sec 6 "doi:10.5281/zenodo.19834321" → new DOI |
| `paper_toms/cover_letter.tex` | reproducibility paragraph "doi:10.5281/zenodo.19834321" → new DOI |
| `paper_toms/reproducibility/acm_artifact_checklist.md` | "Persistent identifier" line → new DOI |
| `CITATION.cff` (repo root) and `paper_toms/zenodo_v2_staging/CITATION.cff` (if present) | `doi:` and `url:` fields → new DOI |

The concept-DOI line in `STATUS.md` should retain the original
concept-DOI (Zenodo concept-DOIs are stable across versions).
